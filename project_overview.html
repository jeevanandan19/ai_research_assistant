Core Features

Everything needed to deeply analyse any research paper

PDF Upload & Processing

Drag-and-drop upload up to 50 MB. PyPDFLoader extracts text page-by-page. RecursiveCharacterTextSplitter creates 1000-char chunks with 200-char overlap for optimal retrieval.

Structured Summary

Generates five sections: Abstract Summary, Detailed Summary, Key Contributions, Limitations, and Future Work — all strictly grounded in the uploaded document.

Keyword Extraction

Extracts and categorises terms into: Primary Keywords, Technical Methods, Datasets & Metrics, and Domain Concepts. Exportable as JSON.

Intelligent Q&A (RAG)

Ask any question. FAISS retrieves the top-K most relevant chunks semantically, then the LLM generates a grounded answer. Falls back to "Information not found" if answer is absent.

Key Insights & Novelty

Extracts core research insights, technical innovation, empirical findings, and broader impact. Independently assesses novelty: High / Medium / Low with justification.

FAISS Vector Store

Each session gets its own FAISS index saved to disk. Supports similarity search with cosine distance. Automatically loaded on each Q&A request.

Chat History

All Q&A interactions are stored per session with timestamps and source counts. Can be cleared or reviewed at any time from the dashboard.

Export

Download the full analysis as a formatted PDF (via ReportLab) or export extracted keywords as a structured JSON file for downstream use.

Gemini Model Fallback

Automatic fallback chain across Gemini models: flash-lite → flash → 2.5-flash → 2.5-pro. If one hits quota (429) or is unavailable (404), the next is tried seamlessly.

Project Workflow

End-to-end RAG pipeline — from PDF upload to AI-generated answer

1
User Uploads PDF

The user drags and drops or selects a research paper (up to 50 MB). The file is validated (type + size), saved securely with a UUID prefix, and a new session is created.

POST /api/upload → werkzeug.secure_filename
2
Text Extraction

PyPDFLoader reads the PDF page-by-page and returns a list of LangChain Document objects, each containing the raw page text and page-number metadata.

PyPDFLoader(file_path).load() → List[Document]
3
Chunking

RecursiveCharacterTextSplitter splits the document into 1000-character chunks with 200-character overlap using semantic separators (paragraphs → sentences → words), preserving context across boundaries.

chunk_size=1000, chunk_overlap=200
4
Embedding Generation

HuggingFace's all-MiniLM-L6-v2 sentence transformer converts every chunk into a 384-dimensional dense vector. Vectors are normalised for cosine similarity. The model is cached locally after first download (~90 MB).

HuggingFaceEmbeddings(model="all-MiniLM-L6-v2")
5
FAISS Indexing

All embedding vectors are inserted into a FAISS flat index and saved to disk under database/faiss_{session_id}/. Each user session gets its own isolated index.

FAISS.from_documents(chunks, embeddings).save_local()
6
User Requests Analysis

The user triggers one or more analysis tasks from the dashboard: Generate Summary, Extract Keywords, Extract Insights, or Ask a Question. Each is an independent API call with results cached in the session.

POST /api/summarize | /api/keywords | /api/insights | /api/ask
7
Semantic Retrieval (RAG)

For Q&A, the user's question is embedded using the same model. FAISS performs a similarity search to return the top-K (default: 5) most semantically relevant chunks from the document.

vector_store.similarity_search(query, k=5)
8
LLM Generation with Custom Prompt

Retrieved chunks are injected into a custom prompt template alongside the user's question. The LLM (Gemini 2.0 Flash or fallback) generates a grounded answer restricted to the retrieved context only.

prompt.format(context=chunks, question=q) → llm.invoke()
9
Response Returned to Dashboard

The answer, retrieved context chunks (with page numbers), and source count are returned as JSON. The dashboard renders the answer in a chat bubble and displays the retrieved chunks in the "Retrieved Context" panel.

{ answer, retrieved_context, num_sources }

Prompt Templates
Summarization Prompt

Generates 5 structured sections. Instructs the LLM to use only provided context and state "Not explicitly mentioned" for absent information.

QA Prompt

Strict retrieval-only answering. Falls back to "Information not found in the uploaded document." if context doesn't contain the answer.

Keyword Extraction Prompt

Returns structured JSON with four categories. Includes regex fallback parser if the LLM doesn't return valid JSON.

Insight Extraction Prompt

Extracts core insight, technical innovation, empirical findings, broader impact, and critical observations in structured markdown.

Novelty Detection Prompt

Rates novelty as High / Medium / Low based only on the paper's own claims. Lists novel contributions and differentiators from prior work.
